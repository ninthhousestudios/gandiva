Visual Companion — What It Can Do

Here are a few examples of what I can show you during our brainstorming. Click any that look useful.

A

UI Mockups & Wireframes

Quick wireframes of layout options — e.g., where the chart style switcher goes, how sidebar widgets dock alongside the chart, how overlay widgets layer on top. Useful for comparing arrangements before writing code.

B

Architecture Diagrams

Visual maps of widget hierarchies, signal/slot relationships, class inheritance trees. E.g., how a ChartRenderer base class relates to WesternWheel, SouthIndianGrid, NorthIndianDiamond subclasses.

C

Side-by-Side Comparisons

Split-screen views comparing design approaches — e.g., "overlay as floating panel" vs "overlay as translucent full-screen layer", or comparing how the South Indian grid vs Western wheel handle the same chart data.

D

Interactive Choices

Clickable option cards like this page — when we reach design decision points, I can present options visually and you can click to select. Faster than describing everything in text.

How it works

I write HTML pages here as we brainstorm. You see them live in the browser. Click options to select them — I can read your clicks. You can also just tell me in the terminal what you think. We'll use whichever is easier for each question.
